### New Framework (Layer 3: Transparent Service Portfolio)
- **Purpose:** Strategic portfolio management layer
- **Focus:** Cost transparency and value optimization
- **Audience:** Executives + finance leaders + portfolio managers
- **Key Concepts:** TBM, service catalog, cost allocation, portfolio optimization

### Why This Reframing Matters

**The Old Narrative:**
"We need to digitally transform government. Here's the executive playbook."

**The New Narrative:**
"Government is a service enterprise. Layer 3 is where we manage that portfolio transparently—every service catalogued, every cost visible, every investment optimized. TBM is how we do it."

**Integration with New Architecture:**

```
Layer 0: Legal & Policy Foundation
    ↓ (enables)
Layer 1: Shared Technology Core
    ↓ (powers)
Layer 2: People & Process Engine
    ↓ (delivers)
Layer 3: Transparent Service Portfolio ← YOU ARE HERE
    ↓ (presents as)
Layer 4: Unified Citizen Experience
```

**Layer 3's Role:**
- Takes services delivered by Layer 2 (People & Process Engine)
- Makes them visible, measurable, and cost-accountable
- Enables strategic portfolio management (invest, optimize, sunset)
- Feeds transparency to Layer 4 (citizens see what they get)

## Key Concepts Introduced in Layer 3

### Technology Business Management (TBM)
- Private sector methodology for IT cost transparency
- Standardized taxonomy (cost towers: Infrastructure, Applications, Data, Security, Labor)
- Cost allocation model linking spending to services
- Benchmarking against industry standards

### Service Portfolio Management
- Treating government as a managed service enterprise
- Every function is a catalogued service with costs, volumes, quality metrics
- Portfolio-level decision making (invest, optimize, sunset)
- Strategic governance, not just operational management

### Radical Transparency
- Publishing full service costs and performance to citizens
- Building trust through demonstrable value
- Enabling democratic debate based on evidence, not rhetoric
- Accountability becomes measurable

### Value Stream Mapping
- Linking services to citizen outcomes
- Connecting costs to strategic objectives
- Measuring operational, citizen, and strategic value
- Portfolio optimization based on value, not politics

## Next Steps for Implementation

### Phase 1: Establish the Catalog (Months 0-6)
1. Inventory all government services (target: 500-1,000)
2. Define service attributes (cost, volume, owner, quality)
3. Create service hierarchy (portfolios → services)
4. Publish initial catalog

### Phase 2: Allocate Costs (Months 6-18)
1. Implement TBM framework (cost towers, allocation drivers)
2. Tag all spending by service
3. Allocate shared Layer 1 platform costs
4. Calculate service costs and cost per transaction

### Phase 3: Publish Transparency (Months 18-24)
1. Build public dashboard
2. Create executive portfolio view
3. Launch benchmarking against peers
4. Communicate value to citizens

### Phase 4: Continuous Improvement (Months 24+)
1. Optimize high-cost services (20%+ over benchmark)
2. Rationalize portfolio (consolidate, sunset low-value)
3. Innovate delivery models (automation, APIs)
4. Expand value metrics (satisfaction, outcomes)
